# 取付管検査 QR閲覧デモ

蓋の裏に貼ったQRコードを読み込むと、その取付管の**検査報告書（PDF）・検査映像・コメント写真集**が
スマホで表示される、静的サイトのデモ一式です。GitHub Pages にそのまま置けます。

> ⚠️ **これはサンプルデータです。** 実現場のデータは含まれていません。
> 展示会・社内でのイメージ共有用にご利用ください。

---

## 構成

```
260701_mapping_qr_demo/
├── index.html              現場一覧（トップページ）
├── vendor/
│   └── qrcode.min.js       QR生成ライブラリ（オフラインで動作）
└── sites/
    └── DEMO001/            1現場ぶんのフォルダ（QRはここを指す）
        ├── index.html      取付管情報＋3ボタン（報告書/映像/写真集）
        ├── report.pdf      検査報告書（現場名〜点検担当者は空欄）
        ├── video.mp4       管内映像サンプル
        ├── photos.html     コメント写真集（スマホ表示用）
        ├── photos.xlsx     コメント写真集（Excel様式・空欄テンプレート）
        └── assets/photos/  写真ファイル
```

## GitHub Pages への公開手順

1. GitHub で新しいリポジトリを作成する（例：`mapping-qr-demo`）。
2. このフォルダの中身を**リポジトリ直下**に置いてコミット・プッシュする。
   （`index.html` がリポジトリのルートに来るようにしてください）
3. リポジトリの **Settings → Pages** を開く。
4. **Build and deployment → Source** を `Deploy from a branch`、
   Branch を `main` / `(root)` にして保存する。
5. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

公開後のURL例：

- 現場一覧　　`https://<ユーザー名>.github.io/mapping-qr-demo/`
- DEMO001　　`https://<ユーザー名>.github.io/mapping-qr-demo/sites/DEMO001/`

---

## QRコードの作り方

各現場ページ（例：`sites/DEMO001/index.html`）を**公開後に開く**と、
ページ内にそのページ自身のURLのQRコードが自動表示されます。
それをスクリーンショット／印刷して蓋の裏に貼ってください。

- 公開前に開くと `file://…` や `localhost` のURLになるため、
  **必ず GitHub Pages 公開後のURLで表示・印刷**してください。

---

## 新しい現場を追加する

1. `sites/DEMO001` フォルダを丸ごとコピーして、`sites/T002` のように短い名前にする。
2. 中の `report.pdf`・`video.mp4`・`photos.xlsx`・`assets/photos/` を差し替える。
3. `photos.html` の写真ファイル名・特記事項ラベルを内容に合わせて直す。
4. トップの `index.html` に現場カードを1つ追加する。
5. コミット・プッシュすると自動で公開されます。

---

## 注意点

- **公開範囲**：GitHub Pages（無料枠）は原則インターネットに公開されます。
  実案件の客先データを載せる場合は、アクセス制御のある別ホスティングを検討してください。
- **ファイルサイズ**：1ファイル100MBが上限（推奨50MB以下）。長い動画は圧縮するか外部に置いてください。
- **QRの読みやすさ**：URLが長いほどQRが密になります。現場フォルダ名は `T002` のように短くしてください。
